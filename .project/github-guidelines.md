# PulseM — GitHub 운용 지침

**버전:** 1.0.0
**최종 수정:** 2026-03-21

---

## 1. 개요

이 문서는 PulseM 프로젝트의 GitHub 저장소 운용 절차를 정의한다. 모든 코드 변경은 이 절차에 따라 진행하며, AI(Claude Code)가 생성한 변경도 동일하게 적용된다.

---

## 2. 브랜치 전략

### 2.1 브랜치 구조

```
main
  └─ feature/#이슈번호-기능-설명      새 기능 개발
  └─ fix/#이슈번호-버그-설명         버그 수정
  └─ refactor/#이슈번호-설명         코드 리팩토링 (PR 불필요)
  └─ docs/설명                       문서 업데이트 (PR 불필요)
  └─ chore/설명                      빌드/설정/의존성 변경 (PR 불필요)
```

### 2.2 브랜치 규칙

- **`main`** 브랜치는 항상 배포 가능한 상태를 유지한다.
- `feat:` / `fix:` 작업은 반드시 브랜치를 만들고 PR을 통해 머지한다.
- `refactor:` / `docs:` / `chore:` 작업은 main에 직접 커밋 허용.
- 브랜치명은 소문자 + 하이픈(-) 사용. 이슈 번호를 반드시 포함.

```bash
# 올바른 브랜치명 예시
git checkout -b feature/#12-upbit-websocket-integration
git checkout -b fix/#34-crypto-price-decimal-display
git checkout -b refactor/market-data-hooks-separation
```

## 3. 이슈 관리

### 3.1 이슈 생성 절차 (`feat:` / `fix:` 작업 시 필수)

PR을 만들기 전 반드시 이슈를 먼저 생성한다.

```bash
# 기능 이슈 생성
gh issue create \
  --title "[feat] 업비트 WebSocket 실시간 코인 시세 연동" \
  --body "## 작업 내용
업비트 WebSocket API를 활용하여 코인 시세를 실시간으로 업데이트한다.

## 수용 기준
- [ ] WebSocket 연결 및 구독 구현
- [ ] 연결 끊김 시 자동 재연결 (최대 3회)
- [ ] 코인 카드 실시간 가격 업데이트
- [ ] 가격 변동 시 틱 애니메이션 표시

---
🤖 Generated by Claude Code [claude-sonnet-4-6]" \
  --label "enhancement,ai-generated"

# 버그 이슈 생성
gh issue create \
  --title "[fix] 코인 가격 소수점 이하 0 표시 오류" \
  --body "## 버그 설명
코인 가격이 소수점 이하 자리수가 있을 때 0이 잘리는 현상

## 재현 방법
1. 코인 탭 이동
2. DOGE 또는 XRP 시세 확인
3. 가격이 0.0X원으로 표시되어야 하나 0.X원으로 표시됨

## 기대 동작
소수점 이하 자리수 정상 표시

---
🤖 Generated by Claude Code [claude-sonnet-4-6]" \
  --label "bug,ai-generated"
```

### 3.2 이슈 라벨 규칙

모든 이슈에 두 종류의 라벨을 **반드시** 조합한다.

**필수 라벨 1:** `ai-generated` (AI가 관여한 모든 이슈)

**필수 라벨 2 (작업 성격, 1개 이상):**

| 라벨 | 의미 |
|------|------|
| `enhancement` | 새 기능 추가 |
| `bug` | 버그 수정 |
| `documentation` | 문서 작업 |
| `refactor` | 코드 리팩토링 |
| `performance` | 성능 개선 |
| `design` | UI/UX 디자인 |
| `security` | 보안 관련 |
| `chore` | 빌드/설정/의존성 |
| `question` | 질문/논의 |

## 4. 커밋 규칙

### 4.1 커밋 메시지 형식

```
{타입}: {한국어 설명}

{선택적 본문 — 변경 이유, 주요 결정 사항}
```

**말머리(타입) 목록:**

| 타입 | 용도 | 예시 |
|------|------|------|
| `feat` | 새 기능 | `feat: 업비트 WebSocket 코인 시세 실시간 연동` |
| `fix` | 버그 수정 | `fix: 코인 가격 소수점 표시 오류 수정` |
| `refactor` | 리팩토링 | `refactor: 시장 데이터 페칭 훅 분리` |
| `docs` | 문서 | `docs: API 스펙 문서 업데이트` |
| `chore` | 빌드/설정 | `chore: Supabase 마이그레이션 파일 추가` |
| `style` | 스타일 포매팅 | `style: Prettier 포매팅 적용` |
| `test` | 테스트 | `test: useStockPrice 훅 단위 테스트 추가` |
| `perf` | 성능 개선 | `perf: 종목 리스트 가상화 적용` |

### 4.2 원자적 커밋 원칙

- 하나의 커밋 = 하나의 논리적 변경
- 여러 기능을 한 커밋에 묶지 않음
- "일단 저장" 스타일의 커밋 메시지 금지

```bash
# 나쁜 예
git commit -m "여러 기능 추가하고 버그도 고침"

# 좋은 예
git commit -m "feat: 주식 시세 카드 컴포넌트 구현"
git commit -m "feat: 주식 시세 30초 폴링 훅 추가"
git commit -m "fix: 시세 카드 등락률 음수 표시 오류 수정"
```

## 5. Pull Request 규칙

### 5.1 PR 생성 절차

```bash
# 1. 이슈 생성 (위 3.1 참고)

# 2. 브랜치 생성
git checkout -b feature/#12-upbit-websocket-integration

# 3. 작업 + 커밋

# 4. 푸시
git push -u origin feature/#12-upbit-websocket-integration

# 5. PR 생성
gh pr create \
  --title "[feat] 업비트 WebSocket 실시간 코인 시세 연동 (#12)" \
  --body "## 작업 내용
업비트 WebSocket API를 통해 코인 시세를 실시간으로 업데이트합니다.

- \`src/components/features/market/CryptoCard.tsx\`: 틱 애니메이션 추가

## 테스트
- [ ] 로컬에서 WebSocket 연결 확인
- [ ] 연결 끊김 후 자동 재연결 동작 확인
- [ ] 가격 변동 시 틱 애니메이션 확인
- [ ] 모바일 뷰 확인
```

### 5.2 PR 제목 형식

```
[{타입}] {한국어 설명} (#{이슈번호})
```

예시:
- `[feat] 업비트 WebSocket 실시간 코인 시세 연동 (#12)`

## 6. 자동 리뷰어 (4명 봇 구성)

### 6.1 리뷰어 목록

| 리뷰어 | 제공사 | 특징 |
|--------|-------|------|
| **Gemini Code Assist** | Google | 코드 정확성, 보안, 성능 분석 |
| **Qodo PR Agent** | Qodo AI | 코드 설명, 개선 제안, 한국어 리뷰 |
| **CodeRabbit** | CodeRabbit AI | 변경 요약, 버그 탐지, 코드 냄새 감지 |
| **GitHub Copilot** | GitHub | 코드 패턴, 베스트 프랙티스 |

### 6.2 리뷰 대응 원칙

1. PR 생성 후 모든 봇 리뷰가 달릴 때까지 기다린다 (보통 5~15분).
2. 각 코멘트를 **검토**한다. 무조건 반영하지 않는다.
3. **보안/버그** 지적 → 최우선으로 검토 후 대응.
4. **성능** 지적 → 프로파일링으로 실제 문제 여부 확인 후 판단.
5. **스타일/취향** 지적 → 프로젝트 컨벤션 기준으로 판단.
6. 동의하지 않는 코멘트 → 이유를 코멘트로 남기고 기각.

### 6.3 리뷰 종합 코멘트 (봇 4개 모두 달렸을 때)

4개 봇 리뷰가 모두 완료되면 Claude Code가 종합 코멘트를 작성한다.

```markdown
## 🤖 리뷰 종합

### 봇 간 공통 지적
- {공통 사항 1}

### 봇 간 의견 차이
- Copilot: {의견} vs CodeRabbit: {의견}

### 채택 사항
- [ ] {수정할 항목}

### 기각 사항
- {기각 항목}: {이유}

@coderabbitai {CodeRabbit에게 질문}
/ask {Qodo에게 질문}
```

## 7. CI/CD 파이프라인

### 7.1 GitHub Actions 워크플로우

```yaml
# .github/workflows/ci.yml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  quality:
    name: 코드 품질 검사
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v3
        with: { version: 9 }
      - uses: actions/setup-node@v4
        with: { node-version: '22', cache: 'pnpm' }
      - run: pnpm install --frozen-lockfile

      - name: TypeScript 타입 검사
        run: pnpm type-check

      - name: ESLint 린팅
        run: pnpm lint

      - name: 단위 테스트
        run: pnpm test --run

      - name: 빌드
        run: pnpm build
        env:
          NEXT_PUBLIC_SUPABASE_URL: ${{ secrets.NEXT_PUBLIC_SUPABASE_URL }}
          NEXT_PUBLIC_SUPABASE_ANON_KEY: ${{ secrets.NEXT_PUBLIC_SUPABASE_ANON_KEY }}
```

```yaml
# .github/workflows/pr-agent.yml — Qodo PR Agent
name: PR Agent
on:
  pull_request:
  issue_comment:

jobs:
  pr_agent:
    runs-on: ubuntu-latest
    if: ${{ github.event.sender.type != 'Bot' }}
    steps:
      - name: PR Agent action
        uses: Codium-ai/pr-agent@main
        env:
          OPENAI_API_KEY: ${{ secrets.OPENAI_API_KEY }}
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          github_action_config.auto_review: "true"
          github_action_config.auto_describe: "true"
          github_action_config.auto_improve: "true"
```

### 7.2 Vercel 배포

| 이벤트 | 동작 |
|--------|------|
| `main` 브랜치 푸시 | 프로덕션 자동 배포 |
| PR 생성/업데이트 | 프리뷰 URL 자동 생성 |
| PR 머지 | 프리뷰 환경 자동 삭제 |

### 7.3 PR 머지 조건

- [ ] CI 모든 스텝 통과 (type-check, lint, test, build)
- [ ] Vercel 프리뷰 배포 성공
- [ ] 봇 리뷰 코멘트 검토 완료
- [ ] approve 불필요 — 봇 리뷰 확인 후 본인이 머지

## 9. 버전 관리 규칙

### 9.1 시맨틱 버저닝

```
{MAJOR}.{MINOR}.{PATCH}

MAJOR: 호환성 깨지는 변경 (대규모 아키텍처 변경)
MINOR: 하위 호환 새 기능 추가
PATCH: 버그 수정
```

MVP 기간은 `0.x.x` 버전 사용. 퍼블릭 런치 시 `1.0.0`.

### 9.2 릴리즈 태그

```bash
git tag -a v0.1.0 -m "MVP 릴리즈: 주식/코인 시세 + 기본 게이미피케이션"
git push origin v0.1.0

# GitHub Release 생성
gh release create v0.1.0 \
  --title "v0.1.0 — MVP 릴리즈" \
  --notes "## 주요 변경사항
- 주식/코인 시세 통합 대시보드
- XP + 레벨 시스템
```
