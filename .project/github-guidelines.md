# PulseM — GitHub 운용 지침

## 1. 이슈 관리
- 모든 작업은 GitHub Issue로 추적
- AI 생성 이슈에는 `ai-generated` 라벨 필수
- 내용에 따라 `enhancement`, `bug`, `documentation`, `refactor` 라벨 추가
- 이슈 제목은 명확하고 구체적으로 작성

## 2. Pull Request 관리
- 모든 코드 변경은 PR을 통해 main에 병합
- 브랜치 네이밍: `feature/#이슈번호-설명`, `fix/#이슈번호-설명`
- PR 본문에 `Closes #이슈번호` 포함
- AI 서명: `🤖 Generated by Claude Code [모델명]`

## 3. 브랜치 전략
- `main`: 항상 배포 가능한 상태
- `feature/*`: 새 기능 개발
- `fix/*`: 버그 수정
- `docs/*`: 문서 개선

## 4. 커밋 규칙
- 한국어 커밋 메시지
- `feat:`, `fix:`, `refactor:`, `docs:`, `chore:` 말머리 사용
- 원자적 커밋 (기능 하나당 커밋 하나)

## 5. 리뷰 프로세스
- 자동 리뷰어: Gemini Code Assist, Qodo PR Agent, CodeRabbit, GitHub Copilot
- CI 통과 필수 (lint, build)
- 봇 리뷰 코멘트 확인 후 머지

## 6. CI/CD
- GitHub Actions: lint → build
- Vercel: main 자동 배포, PR 프리뷰 배포
