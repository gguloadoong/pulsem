# PulseM — Test Plan

## 1. 테스트 전략
| 유형 | 도구 | 범위 |
|------|------|------|
| 정적 분석 | TypeScript strict, ESLint | 전체 코드 |
| 단위 테스트 | Vitest | 유틸리티, 포맷터, 데이터 변환 |
| 컴포넌트 테스트 | React Testing Library | UI 컴포넌트 |
| E2E 테스트 | Playwright | 핵심 사용자 플로우 |
| 성능 테스트 | Lighthouse CI | Core Web Vitals |
| 접근성 테스트 | axe-core | WCAG AA |

## 2. E2E 시나리오
1. **출석 체크인 플로우**: 앱 진입 → 체크인 버튼 클릭 → XP 보상 확인 → 스트릭 증가
2. **카테고리 필터링**: 전체 → 주식 → 코인 → 부동산 탭 전환 시 데이터 필터 확인
3. **뉴스 탐색**: 뉴스 탭 → 뉴스 카드 클릭 → 영향도 표시 확인
4. **하단 탭 전환**: 홈 → 시장 → 뉴스 → 퀘스트 → MY 순차 전환
5. **검색 기능**: 검색 아이콘 클릭 → 검색창 표시 → 키워드 입력

## 3. 엣지케이스
- 네트워크 실패 시 캐시 데이터 표시
- 빈 관심종목 목록 empty state
- 장 마감 시간 외 데이터 표시
- 극단적 가격 변동 (±99%) 표시
- 뉴스 0건일 때 UI 처리
- 연속 출석 365일 이상 시 표시

## 4. 성능 테스트
- FCP < 1.5s, LCP < 2.5s, CLS < 0.1
- 60fps 애니메이션 유지 (마켓 티커, 카드 전환)
- 100개 이상 종목 리스트 렌더링 성능

## 5. 보안 테스트
- XSS: 사용자 입력 검색어 이스케이프 확인
- CSRF: 상태 변경 API 토큰 검증
- 인증: JWT 만료/갱신 시나리오

## 6. 테스트 환경
- 로컬: `npm run dev` + Vitest watch 모드
- CI: GitHub Actions (lint → build → test)
- 브라우저: Chrome, Safari, Firefox (모바일 시뮬레이션)

## 7. QA 자동화 가이드
- PR 생성 시 자동 테스트 실행
- main 머지 후 Lighthouse CI 실행
- 주간 E2E 회귀 테스트 스케줄링
