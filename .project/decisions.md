# PulseM — Architecture Decision Records

## ADR-001: 프로젝트 시작
- **날짜**: 2026-03-21
- **상태**: 승인됨
- **결정**: PulseM 프로젝트를 시작한다
- **맥락**: 주식/코인/부동산 투자정보를 통합 제공하며 게이미피케이션으로 습관 형성
- **결과**: MVP 먼저 구현 후 순차적 기능 확장

## ADR-002: 기술 스택
- **날짜**: 2026-03-21
- **상태**: 승인됨
- **결정**: Next.js 15 + TypeScript + Tailwind CSS v4 + Framer Motion
- **맥락**: 빠른 프로토타이핑, 모바일 퍼스트 UX, 풍부한 애니메이션 필요
- **대안 검토**: React Native (네이티브 앱) → 웹 우선 전략으로 기각
- **결과**: 웹앱으로 시작, PWA → 네이티브 앱 순서로 확장

## ADR-003: 다크 모드 전용
- **날짜**: 2026-03-21
- **상태**: 승인됨
- **결정**: MVP는 다크 모드 전용으로 개발
- **맥락**: 투자 앱 사용자 대다수가 다크 모드 선호, 개발 리소스 절약
- **결과**: Phase 2에서 라이트 모드 지원 추가

## ADR-004: 목데이터 기반 MVP
- **날짜**: 2026-03-21
- **상태**: 승인됨
- **결정**: MVP는 정적 목데이터로 UI/UX 검증 후 실 API 연동
- **맥락**: 외부 API 키 발급/비용 없이 빠르게 UI 프로토타입 완성
- **결과**: Phase 2에서 Supabase + 외부 API 연동

## ADR-005: 이모지 기반 아이콘
- **날짜**: 2026-03-21
- **상태**: 임시
- **결정**: 초기에는 이모지를 아이콘으로 사용
- **맥락**: 별도 아이콘 라이브러리 없이 빠르게 시각적 구분 가능
- **결과**: 디자인 고도화 시 Lucide React로 마이그레이션
