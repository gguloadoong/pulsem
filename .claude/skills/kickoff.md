# Skill: kickoff — 프로젝트 킥오프 미팅

**트리거:** 프로젝트 시작 시 또는 "kickoff 스킬 실행해" 입력 시
**실행 지시:** 각 에이전트가 자신의 관점에서 발언하며 담당 문서를 업데이트합니다.

- Step 1: PM — PRD 초안 발표 (서비스명, 타깃, 핵심 기능) → PRD.md 업데이트
- Step 2: Strategist — 전략 방향(타깃 지표, 포지셔닝) → roadmap.md 업데이트
- Step 3: Designer — 디자인 스펙(톤앤매너, 색상) → design-spec.md 업데이트
- Step 4: FE — 기술 스펙(프레임워크, 아키텍처) → tech-spec.md 업데이트
- Step 5: BE — API/데이터 연동 방안 → api-spec.md, data-sources.md 업데이트
- Step 6: QA — 테스트 플랜 → test-plan.md 업데이트
- Step 7: PM — 종합 정리 → decisions.md, meeting-notes/kickoff.md 저장
