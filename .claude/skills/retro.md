# 레트로 스킬

## 목적
스프린트가 끝날 때마다 팀이 함께 잘된 점과 개선할 점을 정직하게 이야기하고, 다음 스프린트에 반영할 액션 아이템을 만든다. 비난 없음, 판단 없음, 개선만 있음.

## 트리거 조건
- 매 스프린트 종료일 (2주차 금요일 스프린트 리뷰 직후)
- 중요한 장애나 실수 발생 후 (핫 레트로)

---

## 실행 순서

### 사전 준비 (PM, 레트로 시작 전 15분)
- FigJam 또는 Notion 레트로 보드 생성
- 3개 컬럼 준비: 잘됐어요 / 아쉬웠어요 / 시도해볼게요
- 이전 레트로 액션 아이템 이행 여부 확인

### Step 1: 체크인 (5분)
- 오늘 기분/컨디션을 이모지나 단어 하나로 표현
- 무거운 분위기 깨기용, 강요 없음

### Step 2: 이전 레트로 액션 아이템 리뷰 (5분)
- 이전 스프린트 액션 아이템 하나씩 확인
- 완료 / 진행 중 / 미진행 표시
- 미진행 항목은 이번에도 유지 or 삭제 결정

### Step 3: 데이터 리뷰 (5분, PM 진행)
- 스프린트 목표 달성 여부
- 계획 vs 실제 스토리포인트
- 버그 발생 수 및 P0/P1 건수
- 배포 지연 여부

### Step 4: 개인 회고 작성 (7분, 동시에 작성)
- 각자 FigJam/노션에 익명 or 실명으로 카드 작성
- 잘됐어요: 이번 스프린트에서 좋았던 것
- 아쉬웠어요: 불편했거나 개선이 필요한 것
- 시도해볼게요: 다음 스프린트에 해보고 싶은 것

### Step 5: 공유 및 토론 (20분)
- 작성된 카드를 읽고 유사한 것끼리 그룹핑
- 각 그룹에서 중요하다고 생각하는 것 토론
- 방어적 반응 없이 경청
- "왜"를 3번 이상 파고드는 5 Whys 기법 활용

### Step 6: 액션 아이템 도출 (10분)
- 토론에서 나온 개선 아이디어 중 실행 가능한 것 선별
- 각 액션 아이템에 담당자 + 완료 시점 지정
- 최대 3개 (너무 많으면 하나도 안 됨)
- 추상적 액션 금지: "소통 잘하자" ❌ → "PR 리뷰 24시간 내 완료 규칙 도입" ✅

### Step 7: 체크아웃 (3분)
- 한 단어 또는 한 문장으로 오늘 레트로 소감
- 레트로 자체에 대한 피드백도 환영

---

## 레트로 포맷 옵션

### A. 표준 형식 (Start/Stop/Continue)
- Start: 시작해야 할 것
- Stop: 중단해야 할 것
- Continue: 계속해야 할 것

### B. 4L 형식
- Liked: 좋았던 것
- Learned: 배운 것
- Lacked: 부족했던 것
- Longed for: 바랐던 것

### C. Mad/Sad/Glad
- Mad: 화났거나 답답했던 것
- Sad: 아쉬웠던 것
- Glad: 기뻤던 것

---

## 퍼실리테이션 원칙

**Prime Directive (필수 낭독):**
> "우리는 각자가 그 당시 알고 있던 것, 보유한 기술, 가용한 리소스, 처한 상황에서 최선을 다했다고 믿습니다. 이 레트로는 비난이 아닌 개선을 위한 자리입니다."

**금지 행동:**
- 특정 개인을 대상으로 한 비난
- 방어적 해명
- "그건 제 잘못이 아니에요"
- 회고 중 코딩 or 딴짓

## 출력 형식

```markdown
# 레트로: 스프린트 [N]
**일시:** YYYY-MM-DD
**진행자:** 박서준
**참석자:** [이름 목록]

## 스프린트 데이터
- 목표 달성: ✅ 완전 달성 / ⚠️ 부분 달성 / ❌ 미달성
- 스토리포인트: 계획 X pt → 완료 X pt (X%)
- 배포: 정시 / X일 지연
- 버그: P0 X건, P1 X건

## 이전 액션 아이템 리뷰
- [x] [완료된 액션]
- [ ] [미완료 → 이번에도 유지]

## 잘됐어요
- [내용]
- [내용]

## 아쉬웠어요
- [내용]
- [내용]

## 시도해볼게요
- [내용]

## 이번 스프린트 액션 아이템 (최대 3개)
1. [액션] — 담당: [이름] — 완료: [날짜/스프린트]
2. [액션] — 담당: [이름] — 완료: [날짜/스프린트]
3. [액션] — 담당: [이름] — 완료: [날짜/스프린트]
```

---

## 규칙
- 레트로 없이 다음 스프린트 시작 금지
- 액션 아이템은 최대 3개, 반드시 담당자 지정
- 이전 스프린트 액션 아이템 미이행 시 원인 반드시 논의
- 레트로 내용은 팀 내부 문서로만 (외부 공개 금지)
