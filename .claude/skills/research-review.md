# 리서치 리뷰 스킬

## 목적
수행된 시장 분석, 기술 조사, 경쟁사 벤치마크, 사용자 리서치 결과를 팀이 함께 검토하고 실행 가능한 인사이트로 전환한다.

## 트리거 조건
- 분기별 시장/경쟁사 분석 완료 시
- 신규 기능 개발 전 사용자 리서치 완료 시
- 기술 스파이크(새 라이브러리/API 조사) 완료 시
- 경쟁사의 주요 업데이트 발생 시
- 외부 투자자 또는 파트너 미팅 전 시장 검토 시

---

## 리서치 유형별 리뷰 방법

### A. 시장/경쟁사 리서치 리뷰

**진행자:** 전략덕후 김하늘
**참석자:** PM + 관련 팀원
**소요 시간:** 45-60분

**리뷰 구조:**
1. **리서치 범위 확인 (5분)**: 어떤 질문에 답하기 위한 리서치였는가?
2. **핵심 발견 공유 (15분)**: Top 5 인사이트 발표 (데이터 기반)
3. **시사점 토론 (15분)**: PulseM에게 의미하는 것은?
4. **기회/위협 도출 (10분)**: SWOT 또는 기회 매트릭스 작성
5. **액션 아이템 (10분)**: 인사이트 → 구체적 액션

**리뷰 시 핵심 질문:**
- 이 데이터의 출처는 신뢰할 수 있는가?
- 우리 가정과 다른 발견이 있는가?
- 즉시 반영 가능한 인사이트는 무엇인가?
- 추가 검증이 필요한 가설은 무엇인가?

### B. 사용자 리서치 리뷰

**진행자:** PM 박서준
**참석자:** 전체 팀
**소요 시간:** 60분

**리뷰 구조:**
1. **리서치 설계 확인 (5분)**: 방법론, 참여자 프로필, 질문 목록
2. **원데이터 공유 (10분)**: 인터뷰 클립, 설문 차트, 사용성 테스트 영상
3. **패턴 찾기 (20분)**: 반복되는 언급, 공통 불편함, 의외의 발견
4. **인사이트 정리 (15분)**: "사용자는 [상황]에서 [니즈]를 가지고 있다"
5. **기능 연결 (10분)**: 인사이트 → 기능 아이디어 or PRD 수정

### C. 기술 스파이크 리뷰

**진행자:** FE 정민우 또는 BE 최영진
**참석자:** 개발팀 + PM
**소요 시간:** 30분

**리뷰 구조:**
1. **조사 배경 (3분)**: 왜 이 기술을 조사했는가?
2. **조사 결과 (15분)**: 성능, 번들 크기, 학습 곡선, 생태계 활성도
3. **채택/기각 추천 (7분)**: 근거와 함께
4. **다음 단계 (5분)**: 채택 시 POC 계획 or 기각 시 대안

## 실행 순서

### Step 1: 리서치 문서 사전 공유
- 리뷰 48시간 전에 리서치 문서 Notion 공유
- 참석자 사전 읽기 요청
- 핵심 질문 3개 미리 준비해서 오기

### Step 2: 리뷰 세션 진행
- 진행자가 리서치 요약 발표 (10분 이내)
- 참석자 질문 + 토론
- 인사이트 → 액션 전환에 집중

### Step 3: 인사이트 분류

```
즉시 반영 가능 (이번 스프린트):
  - [인사이트] → [구체적 액션]

단기 반영 (1-2 스프린트):
  - [인사이트] → [구체적 액션]

장기 고려 (로드맵):
  - [인사이트] → [방향성]

추가 검증 필요:
  - [가설] → [검증 방법]
```

### Step 4: 액션 아이템 GitHub 등록
- 즉시 반영 항목 → GitHub Issue 생성 + 스프린트 배정
- 단기 항목 → 백로그 등록
- 장기 항목 → 로드맵 문서 업데이트

## 규칙
- 데이터 없는 인사이트는 "가설"로 명확히 표기
- 리서치 결과 기반 액션 아이템은 48시간 내 GitHub 등록
- 기존 팀의 가정과 충돌하는 발견은 별도 토론 시간 마련
- 리서치 원데이터는 Notion에 최소 6개월 보존
