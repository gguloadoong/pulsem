# 루프 스킬

## 목적
특정 태스크를 완료 조건을 충족할 때까지 자율적으로 반복 실행한다. 반복되는 작업(빌드 수정, 테스트 통과, 디자인 반영)을 중단 없이 진행할 때 사용한다.

## 트리거 조건
- 빌드 에러를 모두 해결할 때까지 반복 수정이 필요할 때
- 테스트가 전부 통과할 때까지 코드를 반복 수정해야 할 때
- 디자인 피드백을 모두 반영할 때까지 반복 수정할 때
- QA 버그 수정을 완료 처리할 때까지 반복할 때
- 특정 지표/조건을 달성할 때까지 반복 실험이 필요할 때

---

## 실행 순서

### Step 1: 루프 목표 정의
루프를 시작하기 전에 반드시 명확히 정의한다:
- **완료 조건(Exit Condition)**: 어떤 상태가 되면 루프를 종료하는가?
- **최대 반복 횟수(Max Iterations)**: 몇 번까지 시도하는가? (기본값: 10)
- **타임아웃**: 시간 제한이 있는가?
- **실패 정의**: 어떤 상태가 되면 에스컬레이션하는가?

### Step 2: 반복 실행 구조

```
[루프 시작]
  │
  ▼
현재 상태 평가
  │
  ├── 완료 조건 충족? → YES → [루프 종료 + 결과 보고]
  │
  └── NO
       │
       ▼
     원인 파악
       │
       ▼
     수정 실행
       │
       ▼
     검증 (빌드/테스트/리뷰)
       │
       ├── 최대 반복 도달? → YES → [에스컬레이션]
       │
       └── NO → [현재 상태 평가]로 돌아감
```

### Step 3: 각 반복(Iteration) 내 진행

**Iteration N 시작 시:**
1. 현재 상태 스냅샷 기록 (에러 목록, 실패 테스트 목록 등)
2. 가장 임팩트 큰 문제부터 수정
3. 수정 후 검증 실행
4. 결과 기록 (해결된 것, 남은 것)

**진척 추적:**
- 각 반복에서 해결된 항목 수 기록
- 진척이 없는 반복이 3번 연속이면 접근 방식 전환
- 반복당 최소 1개 이상 해결되어야 정상 진행

### Step 4: 루프 종료 처리

**정상 종료 (완료 조건 충족):**
- 완료 선언 + 최종 상태 보고
- 총 반복 횟수 및 소요 시간 기록
- 주요 변경사항 요약

**비정상 종료 (최대 반복 도달 or 진척 없음):**
- 에스컬레이션: 담당 팀원 또는 CEO에게 보고
- 현재까지 진행 상황 + 막힌 지점 상세 설명
- 추가 도움 필요 사항 명시

## 사용 예시

### 예시 A: 빌드 에러 해결 루프
```
완료 조건: TypeScript 에러 0개, ESLint 경고 0개
최대 반복: 15
실패 정의: 같은 에러가 3번 반복 시 에스컬레이션

Iteration 1: 에러 23개 발견 → 타입 관련 15개 수정 → 에러 8개
Iteration 2: 에러 8개 → import 경로 문제 5개 수정 → 에러 3개
Iteration 3: 에러 3개 → 나머지 수정 → 에러 0개
[완료 조건 충족, 루프 종료]
```

### 예시 B: QA 버그 수정 루프
```
완료 조건: P0, P1 버그 0개
최대 반복: 스프린트 내 (5 영업일)
실패 정의: P0 버그가 5일 이상 미해결 시 에스컬레이션

매일 QA 보고 기반으로 버그 수정 → QA 재검증 → 반복
```

## 출력 형식

```markdown
# 루프 실행 보고

**태스크:** [루프 대상 작업]
**완료 조건:** [조건 명시]
**시작:** YYYY-MM-DD HH:MM
**종료:** YYYY-MM-DD HH:MM

## 반복 기록
| 회차 | 시작 상태 | 수행 작업 | 종료 상태 |
|------|-----------|-----------|-----------|
| 1 | 에러 23개 | 타입 에러 수정 | 에러 8개 |
| 2 | 에러 8개 | import 수정 | 에러 3개 |
| 3 | 에러 3개 | 나머지 수정 | 에러 0개 |

## 결과
**상태:** 정상 완료 ✅ / 에스컬레이션 🚨
**총 반복:** X회
**완료 조건 충족:** YES / NO

## 주요 변경사항 요약
- [변경 1]
- [변경 2]
```

---

## 규칙
- 완료 조건 없이 루프 시작 금지
- 최대 반복 횟수는 반드시 설정 (무한 루프 방지)
- 3번 연속 진척 없으면 접근 방식 전환 또는 에스컬레이션
- 루프 중 스코프 확장 금지 (완료 조건만 충족)
