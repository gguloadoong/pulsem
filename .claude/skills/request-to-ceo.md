# CEO 보고/요청 스킬

## 목적
팀 레벨에서 결정하기 어려운 사안, 예산/방향성/전략 관련 의사결정, 중요한 이슈를 CEO에게 간결하고 명확하게 보고하거나 결정을 요청한다.

## 트리거 조건
- 팀 예산 범위를 초과하는 의사결정
- 제품 방향성 전환이 필요할 때
- 스프린트 목표 달성이 불가능할 때
- 외부 파트너십 계약/협약 관련
- P0 장애 발생 및 해소 보고
- 팀 인력 충원 or 구조 변경 필요 시
- 법률/규제 리스크 발견 시

---

## 보고 유형별 가이드

### A. 의사결정 요청 (가장 흔한 유형)
CEO가 YES/NO 또는 옵션 선택을 해줘야 할 때

**필수 포함 요소:**
1. 결정이 필요한 이유 (배경 1-2줄)
2. 옵션 목록 (최소 2개, 최대 3개)
3. 각 옵션의 트레이드오프
4. 팀 추천 옵션 + 이유
5. 결정 필요 데드라인

### B. 상황 보고 (정보 공유 목적)
CEO가 알아야 하지만 즉각적 결정이 필요 없을 때

**필수 포함 요소:**
1. 무슨 일이 일어났는가 (사실)
2. 현재 영향 (범위, 규모)
3. 팀이 취한 조치
4. 예상 결과 또는 다음 단계

### C. 긴급 에스컬레이션 (P0 이슈)
즉각적인 CEO 개입이 필요한 위급 상황

**즉시 전달 방법:**
- Slack DM + 전화 (순서대로)
- 메시지 시작: 🚨 [긴급] 표시

---

## 실행 순서

### Step 1: 보고 전 자가 검토
보고 전 반드시 확인:
- [ ] 팀 내에서 해결 불가능한 사안인가?
- [ ] 데이터와 사실에 기반하는가?
- [ ] 팀 추천안이 있는가?
- [ ] 결정 데드라인이 명확한가?

### Step 2: 보고 문서 작성
BLOT(Bottom Line On Top) 원칙:
- 첫 문장에 핵심 요청/보고 내용 제시
- 세부 내용은 뒤에 배치
- 총 분량: 200단어 이내 (긴급 시 더 짧게)

### Step 3: 전달 채널 선택
```
즉시 결정 필요 (당일 내) → Slack DM + 보고 문서 링크
일반 보고 (3일 이내) → Notion 문서 + Slack 공유
주간 보고 → 주간 미팅 어젠다에 추가
긴급 (P0) → 전화 + Slack DM 동시
```

### Step 4: CEO 응답 후 처리
- 결정 사항 회의록 or Slack에 기록
- 팀 전체에 결정 내용 공유
- 해당 GitHub 이슈/PRD에 결정 반영
- 필요 시 스프린트 조정

## 출력 형식

### 의사결정 요청 템플릿
```markdown
## [결정 요청] [제목]

**요청자:** [이름 / 역할]
**결정 필요 시한:** YYYY-MM-DD

### 배경
[왜 이 결정이 필요한가 — 2줄 이내]

### 옵션

**옵션 A: [제목]**
- 내용: [1-2줄]
- 장점: [핵심 1-2가지]
- 단점: [핵심 1-2가지]
- 예상 비용/공수: [있으면]

**옵션 B: [제목]**
- 내용: [1-2줄]
- 장점: [핵심 1-2가지]
- 단점: [핵심 1-2가지]
- 예상 비용/공수: [있으면]

### 팀 추천
**옵션 [A/B] 추천**
이유: [핵심 근거 1-2줄]

### 결정 후 다음 단계
- [결정 A 시]: [이후 액션]
- [결정 B 시]: [이후 액션]
```

### 상황 보고 템플릿
```markdown
## [상황 보고] [제목]

**보고자:** [이름 / 역할]
**보고 시각:** YYYY-MM-DD HH:MM

### 현황
[무슨 일이 일어났는가 — 3줄 이내]

### 영향
[어떤 영향이 있는가 — 범위, 규모]

### 팀 조치
[이미 취한 조치 또는 진행 중인 조치]

### 다음 단계
[앞으로 어떻게 진행할 것인가]

### CEO 확인 필요 사항
[있으면 명시, 없으면 "정보 공유 목적" 표시]
```

### 긴급 에스컬레이션 템플릿
```
🚨 [긴급] [제목]

발생: YYYY-MM-DD HH:MM
현상: [한 줄]
영향: [사용자/서비스 영향]
현재 조치: [진행 중인 것]
필요 지원: [CEO에게 필요한 것]
```

---

## 규칙
- 팀 내 해결 가능한 사안을 CEO에게 에스컬레이션 금지
- 보고 전 반드시 팀 추천안 포함
- 보고서에 "모르겠어요" 없음 — 대신 "데이터 부족으로 추가 조사 필요"
- P0 긴급 에스컬레이션은 30분 이내에 첫 보고
- CEO 결정 내용은 반드시 팀 전체에 즉시 공유
