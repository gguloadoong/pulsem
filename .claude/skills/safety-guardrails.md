# 안전 가드레일 스킬

## 목적
PulseM이 금융 정보 서비스로서 법적, 윤리적 경계 안에서 운영되도록 명확한 규칙과 가이드라인을 제공한다. 위반 시 회사 전체가 위험에 처할 수 있다.

## 트리거 조건
- 새 기능이 투자 정보/추천과 관련될 때
- 콘텐츠 생성 AI 기능 개발 시
- 사용자 데이터 수집/활용 기획 시
- 외부 데이터 파트너십 논의 시
- 규제 기관 관련 이슈 발생 시

---

## 금지 행위 (절대 금지)

### 투자 자문 관련
```
❌ 특정 종목 매수/매도 직접 추천
   ("삼성전자 지금 사세요", "이 주식 팔아야 해요")

❌ 수익 보장 또는 손실 보장 표현
   ("이 전략으로 수익 보장", "절대 안 잃는 방법")

❌ 미래 주가 단정적 예측
   ("다음 달 30% 오릅니다", "확실히 바닥입니다")

❌ 투자 자문업 등록 없이 투자 자문 서비스 제공
   (개인화된 투자 포트폴리오 구성 서비스)
```

### 데이터 & 개인정보
```
❌ 동의 없는 개인정보 수집/활용
❌ 금융 거래 내역을 제3자에게 식별 가능한 형태로 제공
❌ 미성년자 금융 투자 정보 타겟 마케팅
❌ 사용자 동의 없는 마케팅 이메일/SMS 발송
```

### 콘텐츠
```
❌ 검증되지 않은 루머, 가짜 뉴스 유포
❌ 시세 조종 목적으로 해석될 수 있는 콘텐츠
❌ 특정 종목 과도한 편향적 긍정/부정 콘텐츠
❌ 저작권 있는 리포트를 무단으로 전문 게시
```

## 허용 범위 (안전 영역)

### 투자 정보 제공 (허용)
```
✅ 공개된 재무 데이터 시각화 및 요약
✅ 뉴스 및 공시 정보 요약 (출처 명시)
✅ 투자 개념, 용어 교육 콘텐츠
✅ 과거 데이터 기반 역사적 분석 (미래 보장 없음 명시)
✅ 시장 지표 및 매크로 데이터 제공
✅ 다양한 관점의 분석가 의견 모음 (중립적 큐레이션)
```

### 게임화 요소 (허용)
```
✅ 학습 기반 XP 보상 (투자 개념 학습, 뉴스 읽기)
✅ 가상 포트폴리오 시뮬레이션 (실제 돈 거래 없음)
✅ 투자 지식 퀴즈 및 테스트
✅ 커뮤니티 내 투자 아이디어 공유 (자문 아님 명시)
```

## 콘텐츠 안전 가이드라인

### 필수 면책 문구
투자 관련 콘텐츠에는 반드시 포함:
```
"이 콘텐츠는 투자 참고용 정보이며, 투자 권유가 아닙니다.
투자 결정은 본인의 판단과 책임 하에 이루어져야 합니다."
```

### AI 생성 콘텐츠 표시
```
"이 요약은 AI가 생성한 내용으로, 원문 확인을 권장합니다."
```

### 표현 가이드

**금지 표현:**
- "지금 사야 한다", "팔아야 한다"
- "확실히", "반드시", "절대적으로" (예측 문맥)
- "수익 보장", "안전한 투자"
- "X씨 추천 종목"

**권장 표현:**
- "일부 애널리스트들은 ... 전망한다"
- "과거 데이터에 따르면 ..."
- "여러 관점 중 하나로 ..."
- "투자 판단 전 다양한 정보를 참고하세요"

## 데이터 처리 안전 기준

### 개인정보 수집 최소화 원칙
```
수집 가능:
  - 이메일 (인증 목적)
  - 닉네임 (서비스 내 표시)
  - 투자 관심 종목 (서비스 개인화)
  - 앱 사용 행동 데이터 (익명화)

수집 불가 (동의 없이):
  - 실제 보유 종목/수량
  - 실제 투자 금액
  - 증권 계좌 정보
  - 주민등록번호, 휴대폰 번호 (필수 서비스 외)
```

### 데이터 보존 기간
```
사용자 계정 데이터: 탈퇴 후 30일 내 삭제
로그 데이터: 최대 1년
분석용 익명화 데이터: 제한 없음
```

## 규제 대응 프로세스

### 규제 이슈 발견 시
```
1단계: 발견 즉시 개발 중단
2단계: CEO + 법무 담당자에게 즉시 보고
3단계: 영향 범위 파악 (어떤 기능, 얼마나 많은 사용자)
4단계: 법무 자문 후 수정 방향 결정
5단계: 수정 완료 후 재배포
```

### 사용자 신고/민원 대응
```
응답 시간: 영업일 기준 3일 이내
에스컬레이션: 투자 손실 관련 민원 → 즉시 CEO 보고
기록 보존: 모든 민원 및 대응 내용 3년 보존
```

## 보안 가드레일

### 인증/인가
- 모든 사용자 데이터 API에 JWT 인증 필수
- RLS로 본인 데이터만 접근 가능
- 관리자 기능에 별도 역할 기반 접근 제어

### 금융 데이터 보안
- 외부 금융 API 키는 서버 사이드에서만 사용 (클라이언트 노출 금지)
- 시세 데이터 캐싱은 공개 데이터만 (개인 계좌 데이터 캐싱 금지)
- 모든 외부 API 통신 HTTPS 필수

## 출력 형식

규제/안전 이슈 발견 시 즉시 보고:

```markdown
# 안전 가드레일 이슈 보고

**발견자:** [이름]
**발견 일시:** YYYY-MM-DD HH:MM
**이슈 유형:** 규제 위반 / 보안 취약점 / 개인정보 / 콘텐츠 안전
**심각도:** 높음(즉시 중단) / 중간(수정 필요) / 낮음(모니터링)

## 이슈 내용
[무엇이 문제인가 — 구체적으로]

## 영향 범위
[어떤 기능, 어떤 사용자, 얼마나 많이]

## 즉각 조치
[개발 중단 여부, 임시 조치 내용]

## 권장 해결 방향
[기술적/정책적 해결 방법]

## 보고 대상
- [ ] CEO 보고 완료
- [ ] 법무 자문 요청 (필요 시)
```

## 규칙
- 투자 권유로 해석될 수 있는 기능은 법무 검토 필수
- 안전 가드레일 이슈는 팀 내 자체 해결 금지 — 반드시 CEO 보고
- 신규 데이터 수집 기능은 개인정보처리방침 업데이트 필수
- 보안 취약점 발견 시 패치 전까지 해당 기능 비활성화
- 이 문서는 분기마다 법무 자문과 함께 업데이트
