# 에이전트 생성 스킬

## 목적
PulseM 팀에 새로운 역할의 에이전트를 추가할 때, 기존 에이전트 파일과 일관된 구조와 품질로 에이전트 파일을 생성한다.

## 트리거 조건
- 새로운 역할이 필요해서 에이전트를 추가할 때
- 기존 에이전트 역할을 분리하거나 전문화할 때
- 외부 협력자(파트너사 담당자 등)를 에이전트로 모델링할 때

---

## 실행 순서

### Step 1: 에이전트 역할 정의
다음 질문에 답하며 에이전트 프로필을 구체화한다:

1. **이름과 별명**: 실제 이름 + B급 감성 별명 (역할 본질을 반영)
2. **출신/경력**: 어느 실존 핀테크/게임 회사 경험? 구체적 프로젝트?
3. **핵심 역할**: PulseM에서 담당하는 것의 핵심 1문장
4. **성격/태도**: 일하는 방식, 싫어하는 것, 강박적으로 챙기는 것
5. **전문성**: 구체적인 기술, 방법론, 도메인 지식
6. **협업 방식**: 기존 팀원 각각과 어떻게 일하는가?

### Step 2: 파일명 결정
- 역할 영어 약칭으로 소문자 파일명 결정
- 예: 데이터 분석가 → `analyst.md`, 마케터 → `marketer.md`
- 저장 위치: `/Users/bong/pulsem/.claude/agents/[파일명].md`

### Step 3: 에이전트 파일 작성

아래 7개 섹션을 반드시 포함한다:

```markdown
# [역할] "[별명]" [이름]

## Identity
[이름, 별명, 출신, 현재 역할, 성격 설명]

## Expertise
[핵심 역량 목록 — 구체적 기술, 방법론, 도구]

## Communication Style
[말투, 응답 방식, 선호 형식, 응답 템플릿]

## Decision Framework
[의사결정 기준, 우선순위 판단 방법, 거부권 행사 기준]

## Tools
[사용하는 도구 목록과 용도]

## Collaboration
[팀원 각각과의 협업 방식 — 기존 6명 에이전트 모두 포함]

## Rules
[반드시 지키는 원칙 5개 이상]
```

### Step 4: 품질 체크

작성 후 아래 기준으로 검토:

- [ ] 성격 묘사가 생생한가? (구체적 습관, 싫어하는 것 포함)
- [ ] 전문성이 PulseM 컨텍스트에 맞게 특화되어 있는가?
- [ ] 기존 팀원 6명 모두와의 협업 방식이 기술되어 있는가?
- [ ] 응답 템플릿이 실제로 쓸 수 있는 형식인가?
- [ ] 의사결정 기준이 구체적이고 실행 가능한가?
- [ ] 규칙이 추상적이지 않고 행동 가능한가?
- [ ] 기존 에이전트 파일 분량/깊이와 유사한가?

### Step 5: 팀 공지
새 에이전트 추가 후 Slack #team-general에 공지:
- 에이전트 이름 + 별명
- 담당 역할 요약
- 파일 경로

---

## 에이전트 별명 작명 가이드

**좋은 별명 특징:**
- 역할의 본질을 담되 B급 감성
- 외우기 쉽고 부르기 편함
- 팀 내 애정 어린 놀림감이 될 수 있는 것

**예시:**
- PM → 기획충 (기획에 미친 사람)
- 전략가 → 전략덕후 (전략 오타쿠)
- 디자이너 → 픽셀장인 (픽셀 장인)
- FE → 프론트괴물 (프론트엔드 괴물)
- BE → 서버신 (서버의 신)
- QA → 버그헌터 (버그 사냥꾼)

## 출력 형식

생성된 에이전트 파일 경로와 핵심 프로필 요약:

```markdown
## 새 에이전트 생성 완료

**파일:** `/Users/bong/pulsem/.claude/agents/[파일명].md`
**이름:** [이름] ("[별명]")
**역할:** [역할 한 줄 설명]
**핵심 전문성:** [3가지]
**팀 내 포지션:** [기존 팀과의 관계]
```

---

## 규칙
- 기존 에이전트 6개 파일을 반드시 먼저 읽고 일관성 맞춤
- 별명 없는 에이전트 생성 금지
- 실존하지 않는 회사 출신으로 설정 금지 (검증 가능한 실존 회사)
- 협업 섹션에서 기존 팀원 누락 금지
